# 설계 문서

## 1. 인증/보안 설계
- 별도 회원 테이블 없이 **단일 마스터 비밀번호**로 게이트
- 비밀번호는 평문 저장 X → bcrypt 해시를 환경변수(`DASHBOARD_PASSWORD_HASH`)로 보관
- 로그인 성공 시 `iron-session`으로 암호화된 httpOnly 쿠키 발급 (서버 세션 저장소 불필요)
- `middleware.ts`에서 `/` 이하 모든 대시보드 경로를 가드, 세션 없으면 `/login`으로 리다이렉트
- Supabase는 **anon key를 브라우저에 절대 노출하지 않음**. 모든 DB 접근은 Next.js API Route(서버)가 `service_role` key로만 수행 → RLS는 기본 차단(all deny) 정책으로 이중 방어
- `robots.txt`에 `Disallow: /`, 메타 `noindex` 추가로 검색엔진 색인 차단

## 2. DB 스키마 (Supabase Postgres)

```sql
create table todos (
  id uuid primary key default gen_random_uuid(),
  content text not null,
  is_done boolean not null default false,
  position int not null default 0,
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table memos (
  id uuid primary key default gen_random_uuid(),
  title text not null default '',
  content text not null default '',
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

create table posts ( -- 게시판(일기/기록장)
  id uuid primary key default gen_random_uuid(),
  title text not null,
  content text not null default '',
  category text not null default '일반',
  images text[] not null default '{}', -- Supabase Storage public/signed URL 목록
  created_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);

-- 2단계: 구글 캘린더 ICS 캐시
create table calendar_events (
  uid text primary key,       -- ICS의 UID
  title text not null,
  start_time timestamptz not null,
  end_time timestamptz,
  location text,
  synced_at timestamptz not null default now()
);
```

- 모든 테이블 RLS 활성화 + 정책 없음(=기본 전체 차단) → API Route(service_role)만 접근 가능
- Storage 버킷 `post-images` : private 버킷, 서버에서 signed URL 발급 방식

## 4. 화면 구성
1. **로그인**: 비밀번호 입력 필드 1개, 오류 시 인라인 메시지
2. **홈 대시보드**: 카드 그리드 (오늘 할일 요약, 최근 메모 3개, 최근 게시글 3개, [2단계] 오늘 일정)
3. **할일**: 체크리스트, 인라인 추가/완료 토글/삭제
4. **메모**: 데스크톱은 목록+에디터 2단 분할, 모바일은 목록→탭하면 편집 화면 전환
5. **게시판**: 카테고리 필터 탭 → 카드형 목록 → 상세(이미지 갤러리 포함) → 수정/삭제
6. **[2단계] 캘린더**: 월간 그리드, ICS 캐시 데이터 표시(읽기 전용)

- 반응형 브레이크포인트: 모바일(~640px) / 태블릿(~1024px) / 데스크톱
- 다크모드: `prefers-color-scheme` 기준 자동 + 수동 토글 버튼

## 5. 배포 설계
- GitHub 저장소 생성(비공개 리포) → Vercel과 연결 → main 브랜치 push 시 자동 배포
- Vercel 환경변수: `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `SESSION_SECRET`, `DASHBOARD_PASSWORD_HASH`, (2단계) `GOOGLE_CALENDAR_ICS_URL`
- Supabase 프로젝트 1개 생성, 위 SQL로 테이블/버킷 세팅
