# 개인 대시보드 기획서

## 1. 목적
- 나 혼자 쓰는 비공개 개인 대시보드
- 노트북/태블릿/폰 어디서든 브라우저로 접속
- 무료 호스팅 기반, 확장 가능한 구조로 시작

## 2. 사용자
- 단일 사용자 (본인만 사용)
- 향후 필요 시 계정 확장 가능성은 열어두되, 지금은 불필요

## 3. 핵심 기능 (MVP)
1. **로그인**: 마스터 비밀번호 1개로 전체 잠금 (회원가입/아이디 개념 없음)
2. **할일/체크리스트**: 오늘 할 일 추가·완료·삭제
3. **메모/노트**: 자유 형식 텍스트 메모 작성·수정·삭제
4. **게시판 (개인 일기/기록장)**: 글쓰기/목록/상세보기, 카테고리 태그, 이미지 첨부

## 4. 2단계 기능 (MVP 이후)
5. **캘린더**: 구글 캘린더의 비공개 ICS 주소를 서버가 주기적으로 읽어와 대시보드에 표시 (읽기 전용, OAuth 불필요)

## 5. 디자인 톤
- 미니멀 카드형 레이아웃
- 다크모드 지원 (라이트/다크 자동 전환)
- 반응형: 데스크톱/태블릿/모바일 대응

## 6. 기술 스택
- **프론트엔드**: Next.js (React) — Vercel 무료 티어 배포
- **백엔드/DB/인증/스토리지**: Supabase 무료 티어
  - Postgres DB: 할일, 메모, 게시글, 캘린더 캐시 저장
  - Supabase Storage: 게시판 이미지 첨부
  - 인증: 자체 구현한 단일 비밀번호 세션 (Supabase Auth의 anon 정책 + 서버 검증)
- **캘린더 연동**: 서버에서 구글 캘린더 비공개 ICS URL을 fetch → 파싱 → DB 캐시

## 7. 구현 순서
1. 프로젝트 셋업 (Next.js + Supabase 연결 + Vercel 배포 파이프라인)
2. 로그인(비밀번호) 게이트
3. 할일/체크리스트
4. 메모
5. 게시판 (+이미지 업로드)
6. 구글 캘린더 ICS 연동
7. 다듬기 (다크모드, 반응형 QA, 성능/보안 점검)

- 무료 티어 한도 내에서 운영 (Vercel/Supabase 무료 한도 확인 필요)
